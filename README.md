# Анализ дропперских кошельков Ethereum

Итоговый проект по курсу **«Анализ данных на Python»**.

**Дропперский кошелёк** — временный адрес в блокчейне, через который в мошеннических схемах проходят переводы. Задача: найти поведенческие признаки таких адресов и научиться отличать их от обычных (`fraud` / `legitimate`).

Данные — готовый датасет Kaggle с агрегатами по Ethereum-адресам (срок жизни, число транзакций, отправители). Самостоятельно собирать транзакции через API не требуется.

## Что сделано

| Этап | Содержание |
|------|------------|
| **1. Сбор данных** | Загрузка Kaggle, стратифицированная выборка 1000 адресов (500 + 500), реестр `wallet_registry.csv` |
| **2. Предобработка** | Импутация медианой, производные признаки, выбросы (IQR + винзоризация) |
| **3. Анализ** | Гипотезы H1–H4: Mann–Whitney U, критерий Фишера |
| **4. Модели** | Корреляции, логистическая регрессия, KNN (подбор k), метрики и ROC на отложенной выборке |

**Главные выводы:** у fraud-кошельков короче срок жизни (H1), выше отношение входящих к исходящим (H3), при ≥100 отправителей выше доля fraud (H4). Простые модели дают F1 ≈ 0.84–0.87 на test.
